# Дизайн и UX ревью: Supply Master

## Общая оценка

**Плюсы:** тёмная тема, акцент золотой, есть тема и единые отступы. Логика шагов понятна.

**Минусы:** шрифт в body не совпадает с темой; инлайн-стили в App; нет состояний hover/focus у кнопок; длинный итог на шаге 4 без группировки; на мобильных прогресс и кнопки требуют доработки.

---

## 1. Визуальная иерархия и типографика

| Проблема | Решение |
|----------|---------|
| В `index.css` указан `-apple-system...`, в теме — Inter. Фактически используется системный шрифт. | Применить `font-family` из темы к `body` в глобальных стилях. |
| Заголовки шагов одинаковые по весу. | Оставить как есть или добавить лёгкий подзаголовок/описание шага для контекста. |
| Нет единого `line-height` в теме. | Добавить в тему `lineHeight` (например 1.4 для body, 1.2 для заголовков) и использовать в компонентах. |

**Рекомендация:** добавить в тему `lineHeight` и применить Inter к `body` через глобальные стили.

---

## 2. Цвета и контраст

| Элемент | Оценка | Рекомендация |
|---------|--------|--------------|
| Фон `#0F1A2F`, текст белый | Хорошо | — |
| Акцент `#C6A15B` на тёмном | Хорошо | Добавить `accentMuted` для фона кнопки "Назад" при hover. |
| Текст `textMuted` 0.5 | Норма | На очень мелком тексте проверить контраст (WCAG AA). |
| Ошибки `#FF6B6B` | Хорошо | Использовать везде для ошибок валидации. |

---

## 3. Кнопки и интерактивность

| Проблема | Решение |
|----------|---------|
| Кнопки "Назад" / "Далее" в App заданы через `style={{ ... }}`. | Вынести в styled-компоненты: единый вид, hover, active, `focus-visible`. |
| Нет визуального feedback при нажатии. | Добавить `transition` и состояние `:active` (лёгкое scale или затемнение). |
| Кнопка удаления в ProductCard — только иконка "✕". | Оставить, но увеличить область нажатия (padding), добавить `aria-label="Удалить товар"`. |
| "Добавить ещё товар" — хороший призыв к действию. | Добавить иконку/стрелку для ясности. |

**Рекомендация:** создать переиспользуемые `PrimaryButton` и `SecondaryButton` с hover/active/focus.

---

## 4. Формы и поля ввода

| Проблема | Решение |
|----------|---------|
| `CurrencyInput` — хороший focus (бордер + тень). | Добавить лёгкий hover (бордер чуть ярче). |
| Числовые поля: при невалидном вводе нет подсветки. | Опционально показывать красную обводку при ошибке (если будет валидация по полям). |
| Select в Step1/Step3 выглядят едино. | Добавить стрелку-индикатор (кастомный или через background-image), чтобы явно было "выбор". |
| Много полей в ProductCard подряд. | На мобильных рассмотреть сворачивание карточки (только название + итог по позиции) с раскрытием по тапу. |

**Рекомендация:** единый hover для всех инпутов; для Step4 — группировка статей (таможня, комиссии, логистика).

---

## 5. Шаг 4 — Итог

| Проблема | Решение |
|----------|---------|
| Длинный список статей (13+ строк) без группировки. | Разбить на блоки: "Стоимость товара", "Логистика и страхование", "Таможня и сборы", "Комиссии и вознаграждение". |
| Итоговая сумма должна быть главным фокусом. | Увеличить размер шрифта, возможно отдельная карточка или верхний блок. |
| Кнопки "Назад" и "Отправить заявку". | Сделать primary только "Отправить заявку", "Назад" — вторичная. Уже так, но оформить через общие компоненты кнопок. |

**Рекомендация:** добавить в Step4Result группы с подзаголовками и визуально выделить итог.

---

## 6. Прогресс и навигация

| Проблема | Решение |
|----------|---------|
| На узком экране подписи шагов "1. Общие" и т.д. могут налезать. | Сократить до "Общие", "Товары", "Логистика", "Итог" или иконки + короткие подписи. |
| Неочевидно, что шаги кликабельны (если решите сделать переход по шагам). | Пока оставить только индикацию; при добавлении перехода — явный cursor: pointer и hover. |
| Полоска прогресса — ок. | Можно добавить лёгкую тень или свечение для акцента. |

---

## 7. Отступы и сетка

| Проблема | Решение |
|----------|---------|
| Контент и кнопки внизу используют `theme.spacing.lg`. | Единообразно: у контента `padding-bottom`, чтобы кнопки не прилипали к блоку. |
| Максимальная ширина 560px — хорошо для чтения. | Добавить `padding-left/right` для очень узких экранов (safe area). |
| Footer отделён border-top — ок. | Можно чуть увеличить верхний отступ футера. |

**Рекомендация:** в App задать `padding-bottom` у Content, чтобы между последним блоком шага и кнопками был зазор; использовать `env(safe-area-inset-*)` для Telegram WebApp.

---

## 8. Доступность (a11y)

| Рекомендация | Действие |
|--------------|----------|
| Фокус-кольцо | У всех интерактивных элементов — видимый outline или box-shadow при `:focus-visible`. |
| Кнопки | У иконки удаления — `aria-label`. У "Далее"/"Назад" можно оставить текст. |
| Заголовки | Сохранить иерархию h1 → h2 → h3. |
| Цвет не единственный индикатор | Ошибки дублировать текстом (уже есть в валидации). |

---

## 9. Чек-лист внедрения

- [ ] Глобальные стили: `body` с Inter и фоном из темы.
- [ ] Тема: добавить `lineHeight`, при необходимости `accentMuted`.
- [ ] Кнопки навигации в App вынести в styled-компоненты с hover/active/focus.
- [ ] ProgressBar: на мобильных короткие подписи или только полоска с номерами.
- [ ] Step4Result: разбить breakdown на группы с подзаголовками.
- [ ] ProductCard: увеличить область нажатия кнопки удаления, `aria-label`.
- [ ] CurrencyInput / Select: единый hover.
- [ ] Safe area для Telegram (padding-bottom у контейнера при необходимости).

После внедрения этих пунктов интерфейс станет более единообразным, предсказуемым и удобным на мобильных и десктопе.
